Welcome to TissuePlex

A step-by-step guide for first-time users

TissuePlex is a web-based viewer for spatial transcriptomics data. It layers cell positions, gene expression, cell-to-cell signaling, and your own analysis results on top of your tissue morphology image — all viewable in a regular web browser, running entirely on your own computer.

This guide is written for people with no prior computational experience. You do not need to know how to program. You just need to be able to follow steps — and you absolutely can do this.

By the end of this guide you will: have TissuePlex installed and running, your Xenium dataset loaded, cell-cell communication edges visible, custom clustering results overlaid, and a confident understanding of every control in the viewer.

What You'll Need

A Mac or Windows computer

TissuePlex runs in the browser — no special graphics card required for basic use. A reasonably modern laptop is fine.

Your Xenium output folder

The folder produced by the Xenium instrument, containing experiment.xenium, morphology.ome.tif, transcripts.parquet, and related files.

An internet connection

Only for the initial download steps. After installation, TissuePlex runs entirely offline on your local machine.

About 30–45 minutes

For the initial setup. Once everything is installed, launching TissuePlex takes a single command and a few seconds.

Part I

Getting TissuePlex Running

In this part you will install two small programs, download TissuePlex, and get it running on your computer. Most of these steps are one-time-only — after that, launching TissuePlex is just one command.

1

Install Git

Git is a program that lets you download software from the internet and keep it up to date. Think of it as a smarter download button — it remembers exactly what version you have and can update it with a single command later.

On a Mac

Open the Terminal application. You can find it in Applications → Utilities, or search for "Terminal" using ⌘ Space and typing "Terminal".

In the Terminal window, type the following and press Return:

git --version

If you see a response like git version 2.x.x, Git is already installed — skip ahead to Step 2. If a pop-up appears offering to install "Command Line Developer Tools" or "Xcode Command Line Tools", click Install and follow the prompts. This installs Git automatically and takes a few minutes.

On Windows

Go to git-scm.com and click the big download button for Windows. Run the installer. You can accept every default setting by clicking "Next" through every screen. When it finishes, you will have a new program called Git Bash — this is what you'll use for all command-line steps in this guide (use it anywhere the guide says "Terminal").

Tip: To confirm Git is working, open Terminal (Mac) or Git Bash (Windows) and run git --version. You should see a version number printed.
2

Install Docker Desktop

Docker is the engine that runs TissuePlex. Think of it like a sealed bubble — it contains all the software TissuePlex needs (Python, databases, web server) without you having to install any of that yourself. You install Docker once, and Docker handles everything else.

Go to docker.com/products/docker-desktop and download Docker Desktop for your operating system. Install it the same way you would install any other application.

Important: After installing, open Docker Desktop and wait for it to fully start up. You will see a small Docker whale icon in your menu bar (Mac) or system tray (Windows) turn from grey/animated to a solid color. TissuePlex will not work unless Docker Desktop is running first.

Recommended: increase Docker's memory limit

In Docker Desktop, open Settings (gear icon) → Resources. If your computer has 16 GB of RAM or more, set the memory limit to at least 8 GB. This helps TissuePlex process large datasets without slowing down. Then click Apply & Restart.

Tip: You need Docker Desktop to be running every time you use TissuePlex. Consider adding it to your login items (Mac: System Settings → General → Login Items) so it starts automatically.
3

Download TissuePlex

Now you will download TissuePlex from GitHub onto your computer. Open Terminal (Mac) or Git Bash (Windows) and navigate to the folder where you want to keep TissuePlex. For example, to put it in your Documents folder:

cd ~/Documents

Then run:

git clone https://github.com/RaredonLab/TissuePlex.git
cd TissuePlex

This creates a folder called TissuePlex in your Documents folder and downloads all of the code into it. The second command moves your terminal prompt inside that folder.

Tip: To update TissuePlex in the future, open Terminal, navigate back to this folder (cd ~/Documents/TissuePlex), and run git pull, followed by docker compose up --build to rebuild with the latest changes.
4

Launch TissuePlex for the First Time

TissuePlex comes with a small sample dataset so you can verify everything is working before touching your real data. Make sure Docker Desktop is running, then — with your terminal still inside the TissuePlex folder — run:

docker compose up --build

The first time you run this, Docker downloads and assembles everything. This can take 5–15 minutes depending on your internet speed. You will see a lot of text scrolling by — that's normal! When you see messages containing uvicorn running and references to nginx, TissuePlex is ready.

Open your web browser (Chrome or Firefox recommended) and go to:

http://localhost:3000

You should see the TissuePlex viewer load with the sample dataset. Congratulations — TissuePlex is running!

Note: The first time a dataset is opened, TissuePlex needs to process the morphology image — this takes 10–30 seconds. If you see a grey or blank canvas, wait a moment and then refresh the page.

Stopping TissuePlex

When you are done for the day, go back to your terminal and press Ctrl + C to stop the running process, then run:

docker compose down
Tip: On subsequent launches you can use the faster docker compose up (without --build) since nothing has changed. Only add --build when you want to pick up code updates after a git pull.
5

Load Your Xenium Dataset

Now let's point TissuePlex at your actual data. TissuePlex needs a parent folder that contains your dataset folder(s). The layout should look like this:

/Users/yourname/xenium-data/ <— this is your DATA_PATH my-experiment-2024/ <— your dataset folder experiment.xenium <— TissuePlex detects Xenium from this file morphology.ome.tif transcripts.parquet cells.parquet ...

You point TissuePlex at the parent folder (e.g. /Users/yourname/xenium-data/), not at the dataset folder itself. You can have as many dataset folders inside the parent as you like — TissuePlex will list them all and let you switch between them.

On Mac or Linux

DATA_PATH="/Users/yourname/xenium-data" docker compose up --build

On Windows (Git Bash)

DATA_PATH="C:/Users/yourname/xenium-data" docker compose up --build
Important: The path must be absolute — it must start from the root of your computer. Paths like ../my-data will not work. On Windows, use forward slashes (/) rather than backslashes (\) in Git Bash.

TissuePlex auto-detects which platform the data came from based on the files present in each folder:

PlatformRequired file in the dataset folder
Xenium (10x Genomics)experiment.xenium
MERSCOPE (Vizgen)cell_by_gene.csv or cell_metadata.csv
CosMx (Nanostring)a file ending in _tx_file.csv

If none of those sentinel files are present, TissuePlex will not list the folder as a dataset — no error, it just skips it.

Part II

Enriching Your Data

TissuePlex can display more than just the standard Xenium output. In this part you will add two kinds of enriched data: cell-cell communication edges from the NICHESv2 R pipeline, and custom cell metadata such as clustering labels or pseudotime scores from your own analysis. Both are entirely optional — TissuePlex works well without them — but adding this data unlocks the most powerful features of the viewer.

6

Understanding the NICHESv2 Edge Layer

One of TissuePlex's most distinctive features is the ability to visualize cell-cell communication as a spatial graph drawn directly on top of the tissue. Each line in this graph represents a pair of neighboring cells, and the lines can be colored and filtered by ligand-receptor mechanisms (LRMs) — the specific molecular signals that one cell sends to another.

This data comes from NICHESv2, a computational R pipeline developed in the Raredon Lab. NICHESv2 takes your Xenium data and scores every spatially neighboring cell pair for every ligand-receptor interaction present in the dataset, producing a rich, quantitative connectivity map.

The output that TissuePlex reads is a single file called edges.parquet, placed in your dataset folder alongside experiment.xenium. If this file is present, the edge visualization layers appear in the Layer Panel. If it is absent, those layers are simply hidden and everything else works normally.

Note: Running NICHESv2 requires R and several R packages, and can take substantial computational time for large datasets. Ask a lab member for guidance on your first run. The steps below describe the workflow so you know what to expect.
7

Install R and RStudio

R is a free programming language widely used in biology and statistics. RStudio is a friendly graphical application for writing and running R code. You will need both.

Install R

Go to cran.r-project.org and download the latest version of R for your operating system. Install it like any normal application.

Install RStudio

Go to posit.co/download/rstudio-desktop and download the free RStudio Desktop version. Install it after R.

Install the required R packages

Open RStudio. In the Console panel at the bottom, paste the following and press Enter:

install.packages(c("Seurat", "arrow", "dplyr", "devtools"))

This may take several minutes. Each package downloads and installs automatically. Then install the NICHESv2 package by following the instructions from the lab — ask a lab member for the current installation command and any additional steps specific to the version you need.

Tip: You only need to install R and RStudio once. Packages also persist between sessions, so this step is a one-time setup.
8

Run NICHESv2 and Export Edge Data

This step involves loading your Xenium data into R, running NICHESv2 to compute cell-cell communication scores, and exporting the result so TissuePlex can display it. The general workflow looks like this:

library(Seurat)
library(NICHESv2)

# 1. Load your Xenium data into R
xenium_obj <- LoadXenium("/path/to/your/xenium-dataset")

# 2. Run NICHESv2 (ask a lab member for recommended parameters)
niches_result <- RunNICHES(xenium_obj, ...)

# 3. Export for TissuePlex — writes edges.parquet to the dataset folder
export_for_TissuePlex(
  niches_result,
  output_dir = "/path/to/your/xenium-dataset"
)

The export_for_TissuePlex() function writes a file called edges.parquet directly into your dataset folder. After it finishes, your folder structure will look like this:

my-experiment-2024/ experiment.xenium morphology.ome.tif transcripts.parquet cells.parquet edges.parquet <— newly created by NICHESv2
Important: The exact arguments for RunNICHES() depend on your dataset and the current version of NICHESv2. Ask a lab member for guidance on your first run. This step can take significant time for large datasets.

Verify the result

Restart TissuePlex with your DATA_PATH. In the Layer Panel on the right, you should now see Tissue Graph and Directed Edges sections that were not there before. If you do not see them, double-check that edges.parquet is directly inside the dataset folder (not in a subfolder).

9

Add Custom Cell Metadata

Beyond the standard Xenium output, TissuePlex can color cells by any metadata you generate yourself — for example: clustering labels, cell type annotations, pseudotime scores, or gene module scores. You add this by placing a CSV file in a special subfolder called cell-metadata inside your dataset directory. TissuePlex finds it automatically and makes every column available in the color-by dropdown.

A typical R workflow

library(Seurat)

# Load your Xenium data
xenium_obj <- LoadXenium("/path/to/your/xenium-dataset")

# --- Run your analysis here ---
# For example, cluster the cells:
xenium_obj <- NormalizeData(xenium_obj)
xenium_obj <- FindVariableFeatures(xenium_obj)
xenium_obj <- ScaleData(xenium_obj)
xenium_obj <- RunPCA(xenium_obj)
xenium_obj <- FindNeighbors(xenium_obj)
xenium_obj <- FindClusters(xenium_obj, resolution = 0.5)

# Extract the columns you want to export
metadata_to_export <- xenium_obj@meta.data[, c("seurat_clusters")]
# Add as many columns as you like — they will all appear in TissuePlex.

# Create the cell-metadata folder if it does not already exist
dir.create(
  file.path("/path/to/your/xenium-dataset", "cell-metadata"),
  showWarnings = FALSE
)

# Save the file — cell barcodes become the row names automatically
write.csv(
  metadata_to_export,
  file.path("/path/to/your/xenium-dataset", "cell-metadata", "my_clusters.csv")
)
Note: The row names of your data frame must match the cell barcodes in the Xenium output. If you loaded the data using LoadXenium(), the barcodes in xenium_obj@meta.data will already match. Ask a lab member for guidance on choosing clustering parameters appropriate for your dataset.

Your dataset folder structure should now look like this:

my-experiment-2024/ experiment.xenium morphology.ome.tif transcripts.parquet cells.parquet edges.parquet cell-metadata/ <— create this folder in R or manually my_clusters.csv <— your exported metadata

After adding the file, restart TissuePlex. Your metadata columns appear automatically under Cell Segments → Color by → Cell metadata. No configuration is needed.

Tip: You can place as many CSV files as you like in the cell-metadata/ folder. TissuePlex merges them all on the barcode key. Each file can contain different columns.
Part III

Using TissuePlex

Now for the rewarding part. This section walks you through every control in the TissuePlex interface — what each one does and when you'd want to use it.

Interface Overview

When TissuePlex loads you will see four main areas:

  • The Viewer — the large central area where your tissue image and all data layers are displayed. Pan by clicking and dragging; zoom with the scroll wheel (or pinch gesture on a trackpad).
  • The Layer Panel — the panel along the right side. This is where you control everything: which layers are visible, how they are colored, which genes or mechanisms to show, and more.
  • The Annotation Toolbar — a compact toolbar at the top of the viewer, containing tools for drawing regions, measuring distances, toggling split-screen mode, and saving images.
  • Info Panels — small floating panels that appear when you click on a cell or an edge, showing detailed information about that object.

Dataset & Image Picker

At the very top of the Layer Panel you will find two dropdown menus:

  • Dataset — lists all the datasets TissuePlex found in your DATA_PATH folder. Switch between datasets here.
  • Image — if your dataset contains multiple OME-TIFF image files, you can switch between them here. Most Xenium datasets have just one.

Switching datasets reloads all data layers automatically. The first time each image is viewed, TissuePlex builds a tile pyramid which takes 10–30 seconds — just wait and refresh if the canvas looks blank.

Tip: TissuePlex works best in Google Chrome or Firefox. If you experience visual glitches in Safari, try switching to Chrome.

Morphology Image

The morphology image is the tissue photograph that forms the base of the viewer. In the Layer Panel, the Morphology row lets you toggle it on or off and adjust its opacity. Reducing opacity can help other data layers stand out more clearly on a dark background.

The image is streamed as a tile pyramid, so it renders sharply at every zoom level without loading the entire file into memory at once.

Cell Segments

The Cell Segments section controls how individual cells are drawn. Each detected cell is shown as a polygon matching its actual measured boundary.

Visibility and Opacity

There are separate opacity sliders for the cell fill (the solid interior) and the cell outline (the border line):

  • Use the checkbox next to Cell Segments to show or hide cell boundaries entirely.
  • The fill slider adjusts how opaque the interior color is — useful for letting the morphology image show through.
  • The outline slider adjusts how prominent the cell border lines are.

There is also a sample slider that controls what fraction of cell boundaries are loaded from the server at any given time. At low zoom levels this is managed automatically; when you are zoomed in, you can raise the sampling fraction to see more cells in detail.

Tip: Showing outlines at full opacity with fill at very low opacity (5–10%) gives a clean view of cell shape and layout without obscuring the underlying morphology image.

Color By

The Color by dropdown is the most powerful control in this section. It lets you color each cell by a data value.

Off (default)

All cells are shown in a neutral color. Use this when you just want to see cell positions without any data overlay.

Gene Set

Colors each cell by the combined expression of a selected set of genes. Cells that express those genes highly appear at the "hot" end of the color scale; cells that do not express them appear at the "cold" end.

  • The gene set is defined using the same gene filter in the Transcripts section — select your genes of interest there, and "Gene Set" coloring will reflect that selection.
  • Choose a color palette: viridis (yellow→purple), plasma (yellow→magenta), magma (yellow→black), or inferno. Viridis is recommended for colorblind accessibility.
  • A color legend appears below showing the data range. Use the clamp sliders on the legend to focus the color range on a specific window of values — for example, if a handful of outlier cells have very high expression and are washing out the variation in the rest, set a lower maximum to bring out the middle range.

Cell Metadata

Colors each cell by any metadata column — including the supplemental metadata you added to the cell-metadata/ folder.

  • Use the field dropdown to pick which column to display.
  • Continuous columns (numbers like pseudotime or module scores) are shown with a gradient palette and clamp controls.
  • Categorical columns (text labels like cluster names or cell types) are automatically assigned a set of distinct colors, one per category. A legend appears listing all categories.

Transcripts

The Transcripts layer shows individual detected RNA molecules as small colored dots, each colored by the gene it represents. This is one of the most detailed views available.

Note: Because a large dataset can have millions of transcripts, this layer only appears when you are zoomed in enough to see individual cells. If you don't see transcript dots, try zooming in further. A small counter in the panel shows how many transcripts are currently visible.

Gene Filter

By default, transcripts from all detected genes are shown simultaneously, each in its own color. You can narrow this to only the genes you care about:

  • The current status is shown as either all N genes or M / N genes selected.
  • Click select ▼ (or edit ▼ if a filter is already active) to open the gene picker.
  • In the picker you can search by gene name, and check or uncheck any gene.
  • Click all to show all genes (remove the filter), or none to start a fresh selection from scratch.
  • Selected genes also have an ✕ button to remove them individually from the active filter.
  • Click clear at any time to reset back to showing all genes.

You can also import and export gene–color assignments as a CSV file using the corresponding buttons — handy for reproducing a specific color scheme across sessions or sharing it with collaborators.

Tip: The gene selection here is shared with the "Gene Set" cell coloring mode. Select a set of marker genes, then switch Cell Segments to "Gene Set" — you will see which cells express those genes, while the Transcripts layer shows where each individual mRNA molecule is located.

Connectivity Layers

These layers are only visible when edges.parquet is present in your dataset folder (see Step 8). They visualize cell-cell communication as a spatial graph overlaid on the tissue.

Edge Density

The density slider (shown as a percentage between the Tissue Graph and Directed Edges sections) controls what fraction of available edges are rendered at any given time. The default is 10% , which balances visual clarity with rendering performance.

  • At low density (1–10%) you see a sparse, readable graph — good for understanding overall connectivity patterns across the tissue.
  • At high density (50–100%) you see far more edges, revealing densely connected hubs — but rendering may slow down for large datasets.
Tip: Start at 10% to get a feel for the landscape, then increase in specific regions of interest to see more detail.

Tissue Graph

The Tissue Graph is a structural background layer showing which cells are spatial neighbors, without any mechanism-specific information. Each line simply means "these two cells are close enough to potentially communicate." Use this layer to understand the physical layout of the cell-cell neighborhood network.

  • Toggle it on or off with the checkbox.
  • The opacity slider lets you use it as a subtle structural reference at low opacity, or bring it to the foreground at full opacity.

Directed Edges

The Directed Edges layer is the quantitative counterpart to the Tissue Graph. Each arrow represents a directed signal from a sending cell to a receiving cell, with its appearance determined by the ligand-receptor mechanisms (LRMs) active on that edge.

When two cells communicate in both directions (A → B and B → A), both arrows are shown offset slightly from each other — like two lanes of a two-way road — making bidirectional signaling easy to see at a glance.

Style Controls

Directional

The Directional checkbox enables direction-aware rendering: A → B and B → A are drawn as two offset parallel lines rather than overlapping. This is on by default and almost always what you want.

Arrowheads

The Arrowheads checkbox adds directional arrow tips at the receiving end of each edge (only available when Directional is on). When arrowheads are active, two additional controls appear:

  • arrow scale slider — adjusts arrowhead size.
  • Full / Harpoon radio buttons — choose between a solid triangle arrowhead ("Full") or a single-sided chevron ("Harpoon"). Harpoon style mimics the chemistry ⇌ notation and can be cleaner for dense graphs.

Autocrine

The Autocrine checkbox toggles visibility of autocrine signals — cases where a cell sends a signal to itself — rendered as rings. When on, two additional sliders appear:

  • ring r — radius of the autocrine ring.
  • ring w — line width of the ring.

Width and Offset

  • width slider — thickness of all edge lines.
  • offset slider (visible when Directional is on) — how far apart the A → B and B → A lines are from each other.

Minimum Strength Filter

The min strength slider filters out weak edges. Only edges whose total interaction score meets the threshold are displayed. Raising this threshold focuses the view on the strongest, most confident cell-cell communications and can dramatically reduce visual clutter.

Edge Color

The edge color dropdown has three modes:

  • Default (uniform) — all edges in a single neutral color.
  • LRM Set (selected mechanisms) — edges are colored by their total communication score across all currently active LRMs. Edges with stronger, more concentrated communication appear at the "hot" end of the palette. The color scale adapts automatically to the data visible in your current viewport. A palette picker and clamp controls appear below.
  • Metadata column — color edges by a metadata field such as sending cell type or receiving cell type. Continuous fields get a gradient; categorical fields get distinct colors with a legend.

LRM Mechanisms Checklist

This is where TissuePlex becomes particularly powerful for mechanistic exploration. The LRM Mechanisms list shows every ligand-receptor mechanism present in your edges.parquet data, and lets you toggle individual mechanisms on or off.

  • A counter shows how many are currently active (e.g. 488 / 488 active).
  • The all and none buttons quickly select or deselect everything.
  • Use the search box to filter the list by gene name — for example, type "Tgfb" to find all TGF-β mechanisms instantly.
  • Toggle individual mechanisms using their checkboxes. Hiding an LRM reduces each edge's "visible score" by that mechanism's contribution, and may hide edges whose score drops to zero.
Example: To ask "which cells are communicating via TGF-β signaling?", click none to hide all LRMs, then search for "Tgfb" and enable only the TGF-β mechanisms. Only edges driven by TGF-β will remain visible.

Clicking to Explore

You can click on individual cells and edges to see detailed information in floating info panels.

Clicking on a Cell

Click anywhere inside a cell's boundary to open the Cell Info Panel. It shows the cell's barcode ID and all available metadata values for that cell, including any supplemental columns you added. The current color-by value is highlighted to make comparison easy. Click anywhere on the empty canvas to dismiss the panel.

Clicking on an Edge or Tissue Graph Line

Click on any line in the Tissue Graph or Directed Edges layer to open the Edge Info Panel, which shows the sending and receiving cell barcodes and the top-scoring ligand-receptor mechanisms on that edge.

Tip: When edges and cells overlap, a normal click prioritizes the cell. Hold Shift while clicking to prioritize edges instead — useful for inspecting a specific edge that passes through a densely packed region.

Split-Screen Mode

Split-screen mode divides the viewer into two independent side-by-side panels. Each panel can be panned and zoomed to a different region of the tissue — perfect for comparing two areas, or for looking at the same region before and after applying a filter.

  • Click ⊞ Split in the annotation toolbar to enter split-screen mode.
  • Click □ Single to return to a single panel.
  • Both panels share all layer settings (visibility, color-by, LRM filter, etc.), but their pan and zoom positions are completely independent.

Matching Zoom Between Panels

The ⇔ Match button in the toolbar makes the other panel match your current panel's zoom level, while each panel stays centered on its own region. This is useful when you want to compare two areas at precisely the same magnification.

Annotation Tools

The annotation toolbar at the top of the viewer offers three interaction modes — click the button for the mode you want:

  • Pan — the default mode. Click and drag to pan the viewer.
  • ⬡ Region — draw a polygon region on the tissue. Click to place each vertex; double-click to close the polygon and create the region. Regions are colored and labeled, and remain visible on top of all data layers. You can draw as many regions as you like.
  • ⟷ Measure — click two points on the tissue to draw a measurement line and read off the distance between them in micrometers.

The Clear button removes all regions and measurements at once.

Exporting Cells Within a Region

Once a region is drawn, it appears in the Layer Panel's Regions list. Each region has an export button that saves a CSV file containing the barcodes of all cells enclosed by that region. This is useful for extracting a spatially defined subset of cells for downstream analysis in R.

Saving Images

TissuePlex has a built-in image export that composites the morphology image and all visible data layers into a single PNG file.

Click the Save PNG button in the annotation toolbar. TissuePlex will download a PNG named tissueplex_[timestamp].png (or tissueplex_panel1_[timestamp].png / panel2_ in split-screen mode) to your browser's default Downloads folder.

Tips for high-quality output

  • Zoom in before saving — the export captures the viewer at its current resolution, so a more zoomed-in view captures finer detail.
  • Maximize your browser window to get a larger canvas.
  • Draw annotation regions or measurements before exporting if you want to mark up the area of interest for a figure.
  • Reduce opacity on layers you are not featuring so the key layer stands out clearly.
  • For very high-resolution output, use Chrome's built-in full-page screenshot: open DevTools (⌘ Option I on Mac or F12 on Windows), press ⌘ Shift P (Mac) or Ctrl Shift P (Windows), type "screenshot", and select Capture screenshot.
Help

Troubleshooting

ProblemWhat to do
Blank or grey canvas after the dataset loads TissuePlex is still processing the morphology image. Wait 15–30 seconds and refresh the page.
My dataset doesn't appear in the dropdown Check that DATA_PATH points to the parent folder (not the dataset folder itself), and that the dataset folder contains the correct sentinel file (e.g. experiment.xenium).
Tissue Graph and Directed Edges don't appear in the panel edges.parquet is missing or in the wrong place. It must be directly inside the dataset folder, alongside experiment.xenium.
My metadata columns don't appear in the Color-by dropdown Check that your CSV is inside a folder named exactly cell-metadata within the dataset folder, and that the file has row names (barcodes) as its first column — R's default write.csv() behavior handles this automatically.
Transcripts are not showing Transcripts only appear when zoomed in close enough to see individual cells. Try zooming in more. Also confirm the Transcripts toggle is on in the Layer Panel.
TissuePlex is running slowly or edges lag Reduce the density slider to 5% or lower. Also check that Docker Desktop has at least 8 GB of memory allocated (Settings → Resources).
Error: port 3000 is already in use Another application is using that port. Stop the other application, or edit docker-compose.yml and change the host port (e.g. 3001:80), then restart.
Something else is wrong Ask a lab member, or open an issue at github.com/RaredonLab/TissuePlex/issues.
